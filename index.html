🛡️ EDR Tactical Remediation Hub

Policy Implementation

  • • Anti-Tamper: Wajib diatur ke "Strict/On" untuk mencegah penghentian proses agent.
  • • Kernel AI: Aktifkan "Static AI - Malicious/Suspicious" pada semua driver engine.
  • • Deep Visibility: Aktifkan pencatatan event pemuatan driver.

S1QL Hunting Query

IndicatorName = "VulnerableDriverLoaded" OR File.Name Contains "rwdrv" OR (File.Sha256 = "12fcde06ddadf1b48a61b12596e6286316fd33e850687fe4153dfd9383f0a4a0")

Prevention Policy Hardening

  • • Vulnerable Driver Protection: Enable (Pastikan settingan ini aktif di Prevention Policy untuk memblokir BYOVD otomatis).
  • • Sensor Anti-Tampering: Set ke "Aggressive".
  • • Custom IOA: Buat IOA untuk mendeteksi penulisan file .sys oleh proses non-SYSTEM.

Falcon Query (LQR)

event_simpleName=DriverLoad AND FileName="rwdrv.sys"

KQL Advanced Hunting

DeviceImageLoadEvents | where FileName =~ "rwdrv.sys" or SHA256 == "16f83f056177c4ec24c7e99d01ca9d9d6713bd0497eeedb777a3ffefa99c97f0" | project Timestamp, DeviceName, FolderPath

👤 Identity & Account Visibility

Qilin biasanya masuk melalui VPN dan RDP menggunakan akun domain. Strategi pembatasan dan pemantauan sangat krusial.

Mitigasi Akun
  • • Batasi Login RDP hanya untuk grup admin tertentu melalui GPO.
  • • Terapkan MFA (Multi-Factor Authentication) pada semua VPN & Remote Access.
  • • Pantau anomali jam login dan lokasi IP yang tidak biasa pada akun domain.
Account Visibility

Gunakan tools AD Audit atau Azure AD Identity Protection untuk mendapatkan visibilitas aktivitas akun secara real-time.

🔍 Proactive Hunting Strategy

Jangan hanya menunggu deteksi; lakukan pemantauan aktif terhadap file yang dieksekusi di lingkungan kerja.

Execution Monitoring
  • • Cari file `.sys` baru yang muncul di direktori `\Temp` atau `\AppData`.
  • • Hunt proses `msimg32.dll` yang dimuat dari direktori selain `\System32`.
  • • Audit log eksekusi `EDRKiller` melalui Sysmon (Event ID 1).
Proactive Rule

Buat rule alert jika ada proses yang mencoba menulis ke \Device\PhysicalMemory, yang merupakan ciri khas manipulasi kernel oleh BYOVD.
